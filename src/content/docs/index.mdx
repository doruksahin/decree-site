## Code outlives the reasons for it

A decision gets made, the code gets written, and the reasoning evaporates. Humans
forget. Agents never knew. By the time a change ships, nobody can say which decision
it was supposed to honor — so the next edit quietly breaks it.

decree keeps the decision attached to the code, and checks every change against it.

## Two exit codes are the whole contract

decree is built to run in CI and in agent loops, so its verdict is the exit code —

## Built for agents, not just people

An LLM agent editing your code has no memory of the decisions it's about to break.
decree gives it a deterministic check it can run before and after every edit: map the
file, check the plan, review the diff. The output is structured JSON; the exit code
gates the loop. The same command that helps a human in review keeps an agent inside
the lines.

## Deterministic, not “AI”

decree has no model and no temperature. The same corpus and the same diff produce the
same answer, every run. It reads declared facts and computes structural overlap — it
never asks a language model what your code probably means. That's what makes its
output safe to gate CI on.

## Trustworthy because it abstains

decree's signals are honest about their own limits, and the boundary is the point.
